## [0.2.0] тАФ 2026-05-13 тАФ "Hermes"

### Added

- **FR-ZSN2GG** тАФ Quack ships as a Claude Code marketplace plugin in the same repo as the server. `.claude-plugin/marketplace.json` at the repo root declares `./plugins/quack/` as the single plugin source. Inside `plugins/quack/`: plugin manifest (`plugin.json`), three thin-shell hook wrappers under `hooks/` that `exec quack-hook <kind> "$@"` against the M3-built binary (silent-disable + one stderr line when binary is not on PATH so a broken install never breaks a Claude Code session), one MCP server declaration (`mcp-servers/quack.json`) with `${QUACK_SERVER_URL:-http://127.0.0.1:7474}` default + `${QUACK_TOKEN}` intentionally-no-default (absence yields a one-time `401` that prompts the user to run `/quack:install`), one `/quack:install <slug>` slash command (slug regex `^[a-z0-9][a-z0-9_-]{0,62}$`; reads `QUACK_ADMIN_TOKEN`; calls admin MCP tools `create_project` / `register_user` / `add_member` with graceful `*_exists` / `already_member` handling; writes a per-workspace `.envrc` for direnv with `chmod 600`; follow-up instructions lead with `unset QUACK_ADMIN_TOKEN`), and a plugin README documenting the full install flow. Server source (`src/`, `compose.yml`, `Dockerfile`, `specs/`, `tests/`) lives outside the plugin source path and is **NOT** included in the marketplace install. Repo-root `README.md` gains a new "Install as Claude Code plugin" section pointing end users at the plugin path; the existing Deployment section is amended for server operators. `.dockerignore` excludes `plugins/` and `.claude-plugin/` so the marketplace surface stays out of the server image. Install command shape: `claude plugin marketplace add ./` + `claude plugin install quack@quack`; installed tree lands at `~/.claude/plugins/cache/quack/quack/<version>/`. Tests: `tests/plugin-version-sync.test.ts` (plugin.json тЖФ marketplace.json parity), `tests/plugin-hooks-syntax.test.ts` (`chmod +x` + `sh -n` + silent-disable marker), `tests/plugin-install-local.test.ts` (source-tree invariants always; opt-in real CLI round-trip via `QUACK_E2E_PLUGIN=1`), `tests/plugin-files.test.ts` (mcp-servers + plugin/repo READMEs + dockerignore shape pins). AC-ZSN2GG.11 (end-to-end manual smoke: real Claude Code session тЖТ `Decision` node in Neo4j тЖТ recall via `search_memory` from a fresh session) is documented in the plugin README and deferred from automation.

### Fixed

- `bun test` integration suites that spawn Neo4j containers in `beforeAll` (under `src/{graph,extract,mcp}/`) no longer time out at Bun's 5000ms default. The `, 180_000` second arg on `beforeAll(...)` is silently ignored тАФ that signature is `test()`-specific тАФ so the project's `test` script now passes `--timeout 180000` and `CLAUDE.md`'s gate command is `bunx tsc --noEmit && bun run test`. The pre-M4 v0.1.0 ship recorded `288 tests, 0 failures, 0 errors` only because the Docker daemon happened to be unreachable at that moment and the integration tests short-circuited via `dockerAvailable()`; the gate is now reliable across both states.

Total test count at release: 314 tests, 0 failures, 0 errors.

## [0.1.0] тАФ 2026-05-13 тАФ "Mnemosyne"

### Added

- **FR-SFQDXR** тАФ GraphAdapter interface + Neo4j Community 5.x wiring. `compose.yml` graphdb service (no profile gate); `quack` depends_on it with `condition: service_healthy`. `getDriver()` singleton with signal-bound shutdown. `Neo4jGraphAdapter.run(templateId, params, ctx)` is the only Cypher entry point; `ctx.project_id` overrides any caller-supplied `project_id` as defense-in-depth. `TEMPLATE_REGISTRY` + `validateTemplateRegistry` enforce the `$project_id` lint at startup. v1 index DDL (11 indexes) idempotent on re-run. `/health` reports `graphdb: "ok" | "down"` via 1-s Bolt probe. Repo-level grep refuses `neo4j-driver` imports outside `src/graph/`. `server_status.counts.graphdb` exposes status + index count.
- **FR-DPY5GQ** тАФ Memory-plane MCP tools (member-readable, returning structured DTOs only, never prose). `search_memory` (full-text + 1-hop expansion), `get_neighbors` (bounded variable-length walk, depth тЙд 3), `path_between` (`shortestPath`, max_hops тЙд 8), `recent_decisions` (time-windowed Decision lookup). Every response carries the canonical `meta` envelope (`mode_used` / `coverage` / `warnings` / optional `explain`) and each `MemoryItem` exposes a `_memory_wrapped` field тАФ the `<memory>тАж</memory>`-tagged serialization is the prompt-injection-laundering boundary. Reserved `mode: "planned"` surfaces `not_implemented_yet` for non-breaking future planner-mode upgrade. Cross-tenant isolation verified via real-Neo4j integration tests.
- **FR-4NY6S1** тАФ Extraction loop. `POST /ingest` validates the `HookEnvelope` (`kind` тИИ session_start / stop / post_tool_use) and enqueues into a bounded FIFO ring buffer; 202 acceptance with 202 `accepted: false` backpressure when full. Consumer task drains at `QUACK_EXTRACTOR_CONCURRENCY` (default 2), runs the shared redaction pass (OpenAI / GitHub / Slack / Bearer / JWT / .env-style defaults) before the model POST, calls an OpenAI-compatible endpoint with strict json_schema + json_object fallback (per-baseURL capability cache), Zod-validates the response, and writes via six no-APOC `MERGE` templates in entity-first order (Entity тЖТ File тЖТ Symbol тЖТ Decision тЖТ Feedback тЖТ Relation). Cross-tenant defense overrides any model-supplied `project_id` with `ctx.project_id`. Failed extractions dead-letter to `${QUACK_DATA_DIR}/dead-letters.jsonl` with size-rotation. `server_status.queue` reports live depth + accepted_total / dropped_full_total counters.
- **FR-EDXH3X** тАФ Pending-cleanup reconciliation sweep + `delete_project` graph cleanup. `delete_project` now writes `pending_cleanup.ref = String(project.id)`; the v2 SQLite migration translates pre-M3 slug-based refs into project_id and drops orphan-slug rows. Single-flight sweeper runs at `QUACK_CLEANUP_INTERVAL_SECONDS` (default 24 h), reads pending rows with `fail_count < 3`, runs batched `MATCH (n {project_id: $project_id}) DETACH DELETE n` until drained; per-row `fail_count++` on Neo4j error; stuck rows (`>= 3`) skipped + logged. Two new admin MCP tools: `run_cleanup_now` (manual trigger; refuses `sweep_in_progress`) and `cleanup_status` (pending_rows / stuck_rows / last_run / currently_running). `server_status` gains a `cleanup` block.
- **FR-S2D0Z5** тАФ `quack-hook` client binary (Claude Code hooks тЖТ POST /ingest). Single self-contained binary built via `bun build --compile src/hooks/quack-hook.ts --compile --outfile dist/quack-hook` (new `bun run build:hook` script). Reads hook payload from stdin, runs the same redaction pattern set as the server (defense-in-depth; `QUACK_HOOK_REDACTION_PATTERNS` env extras append to defaults), fire-and-forget POSTs to `${QUACK_SERVER_URL}/ingest` with a 1 s `AbortSignal.timeout`. Exits 0 on every code path that isn't a CLI usage error тАФ the Claude Code session is never broken by a hook server issue. `quack-hook init <slug>` writes `~/.quack/projects/<slug>.env` + prints a Claude Code hooks-config YAML snippet. `.dockerignore` excludes `dist/`; the compiled binary is a client-side artifact.

Total test count at release: 288 tests, 0 failures, 0 errors.
