## [0.1.0] — 2026-05-13 — "Mnemosyne"

### Added

- **FR-SFQDXR** — GraphAdapter interface + Neo4j Community 5.x wiring. `compose.yml` graphdb service (no profile gate); `quack` depends_on it with `condition: service_healthy`. `getDriver()` singleton with signal-bound shutdown. `Neo4jGraphAdapter.run(templateId, params, ctx)` is the only Cypher entry point; `ctx.project_id` overrides any caller-supplied `project_id` as defense-in-depth. `TEMPLATE_REGISTRY` + `validateTemplateRegistry` enforce the `$project_id` lint at startup. v1 index DDL (11 indexes) idempotent on re-run. `/health` reports `graphdb: "ok" | "down"` via 1-s Bolt probe. Repo-level grep refuses `neo4j-driver` imports outside `src/graph/`. `server_status.counts.graphdb` exposes status + index count.
- **FR-DPY5GQ** — Memory-plane MCP tools (member-readable, returning structured DTOs only, never prose). `search_memory` (full-text + 1-hop expansion), `get_neighbors` (bounded variable-length walk, depth ≤ 3), `path_between` (`shortestPath`, max_hops ≤ 8), `recent_decisions` (time-windowed Decision lookup). Every response carries the canonical `meta` envelope (`mode_used` / `coverage` / `warnings` / optional `explain`) and each `MemoryItem` exposes a `_memory_wrapped` field — the `<memory>…</memory>`-tagged serialization is the prompt-injection-laundering boundary. Reserved `mode: "planned"` surfaces `not_implemented_yet` for non-breaking future planner-mode upgrade. Cross-tenant isolation verified via real-Neo4j integration tests.
- **FR-4NY6S1** — Extraction loop. `POST /ingest` validates the `HookEnvelope` (`kind` ∈ session_start / stop / post_tool_use) and enqueues into a bounded FIFO ring buffer; 202 acceptance with 202 `accepted: false` backpressure when full. Consumer task drains at `QUACK_EXTRACTOR_CONCURRENCY` (default 2), runs the shared redaction pass (OpenAI / GitHub / Slack / Bearer / JWT / .env-style defaults) before the model POST, calls an OpenAI-compatible endpoint with strict json_schema + json_object fallback (per-baseURL capability cache), Zod-validates the response, and writes via six no-APOC `MERGE` templates in entity-first order (Entity → File → Symbol → Decision → Feedback → Relation). Cross-tenant defense overrides any model-supplied `project_id` with `ctx.project_id`. Failed extractions dead-letter to `${QUACK_DATA_DIR}/dead-letters.jsonl` with size-rotation. `server_status.queue` reports live depth + accepted_total / dropped_full_total counters.
- **FR-EDXH3X** — Pending-cleanup reconciliation sweep + `delete_project` graph cleanup. `delete_project` now writes `pending_cleanup.ref = String(project.id)`; the v2 SQLite migration translates pre-M3 slug-based refs into project_id and drops orphan-slug rows. Single-flight sweeper runs at `QUACK_CLEANUP_INTERVAL_SECONDS` (default 24 h), reads pending rows with `fail_count < 3`, runs batched `MATCH (n {project_id: $project_id}) DETACH DELETE n` until drained; per-row `fail_count++` on Neo4j error; stuck rows (`>= 3`) skipped + logged. Two new admin MCP tools: `run_cleanup_now` (manual trigger; refuses `sweep_in_progress`) and `cleanup_status` (pending_rows / stuck_rows / last_run / currently_running). `server_status` gains a `cleanup` block.
- **FR-S2D0Z5** — `quack-hook` client binary (Claude Code hooks → POST /ingest). Single self-contained binary built via `bun build --compile src/hooks/quack-hook.ts --compile --outfile dist/quack-hook` (new `bun run build:hook` script). Reads hook payload from stdin, runs the same redaction pattern set as the server (defense-in-depth; `QUACK_HOOK_REDACTION_PATTERNS` env extras append to defaults), fire-and-forget POSTs to `${QUACK_SERVER_URL}/ingest` with a 1 s `AbortSignal.timeout`. Exits 0 on every code path that isn't a CLI usage error — the Claude Code session is never broken by a hook server issue. `quack-hook init <slug>` writes `~/.quack/projects/<slug>.env` + prints a Claude Code hooks-config YAML snippet. `.dockerignore` excludes `dist/`; the compiled binary is a client-side artifact.

Total test count at release: 288 tests, 0 failures, 0 errors.
