# M10 — Typed MCP tool inputSchema + extraction noise suppression

## M10: Typed MCP tool inputSchema + extraction noise suppression {#M10}

**Goal:** Fix two defects found during the M-series install smoke on a real
schema-driven MCP client. (A) MCP tools advertise an empty passthrough
`inputSchema`, so array-valued args (`search_memory.entities`, …) are mis-encoded
by typed clients and rejected as `invalid_args` — `search_memory` is undrivable.
Advertise per-tool typed `inputSchema` while keeping the handler-level `safeParse`
as the validation authority so `AC-WSFVNP.10` is preserved. (B, layered) The
`PostToolUse` hook forwards the agent's own tool-search activity, which the
extractor mints into low-value `Decision` nodes; drop it at the client hook and
harden the extractor prompt.

**Prerequisites:** M2 (`FR-WSFVNP` — the admin/MCP tool surface + the
`invalid_args` contract this fix must preserve) and M5 (`FR-55S220` — the
`.mcp.json` hook config path the client drop sits beside) shipped.
