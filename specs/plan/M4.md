# Implementation Plan

## Milestone Order

M1 (Bootstrap) shipped. M2 (multi-tenant auth + admin MCP + `server_status` + Docker) shipped. M3 (memory plane — Neo4j + extraction + memory MCP tools + hooks) is the active implementation milestone. M4 wraps Quack as a Claude Code marketplace plugin so installation becomes a one-command experience for end users.

## M4: Plugin packaging (marketplace + thin-shell hooks + MCP config + /quack:install) {#M4}

**Goal:** Quack is installable as a Claude Code plugin via `claude marketplace add ./` + `/plugin install quack`. The plugin ships only `plugins/quack/` (hooks + MCP config + setup command + README); server source, Compose stack, and specs stay invisible to plugin users. End-to-end UX: clone repo → `docker compose up` → `bun run build:hook` + install binary → install plugin → per-workspace `/quack:install <slug>` → start a Claude Code session and the memory pipeline works.

**Prerequisites:** M3 shipped (the `quack-hook` binary from FR-S2D0Z5 must exist for the plugin's shell wrappers to exec; the memory-plane MCP tools from FR-DPY5GQ must exist for the MCP config to do anything useful; the admin MCP tools from FR-WSFVNP must exist for `/quack:install` to mint tokens).
