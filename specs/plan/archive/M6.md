# M6 — Plugin-hermetic hooks

## M6: Move hook execution into the plugin (bunx-driven, no binary) {#M6}

**Goal:** Eliminate the install-side `quack-hook` binary + PATH dance.
Hook code becomes hermetic inside `plugins/quack/`, executed via
`bunx --bun` straight from the plugin tree. Adds a defensive layer
(parseHookPayload lib + bundled-hooks-shape contract test) mirroring
dev-process-toolkit's STE-289 / STE-290 fixes.

**Prerequisites:** M2 (auth + admin MCP), M3 (ingest + binary —
deleted by this FR), M4 (plugin packaging), M5 (add_memory).
