# M9 — Agentic ask_memory tool

## M9: Agentic ask_memory tool {#M9}

**Goal:** Add a fifth memory-plane MCP tool, `ask_memory`, that answers free-form questions
with a bounded server-side agentic retrieve→reason loop. The cheap model (reusing
`QUACK_MODEL_*`) plans retrieval by calling the existing four read primitives internally,
follows a few hops, and returns a synthesized natural-language answer. This is the single,
deliberate exception to the prose-free / "dumb server" read-path split, bounded by
defense-in-depth (no NL→Cypher; redaction before re-prompting; untrusted-data framing;
`<memory>`-wrapped answer). Realizes the `mode: "planned"` enum reserved in AC-DPY5GQ.8.

**Prerequisites:** M8 (cross-project memory) shipped at v0.5.0. Builds on FR-DPY5GQ (the four
primitives + `meta` envelope), FR-41NXTZ (`add_memory` — the existing server-side LLM tool),
and FR-4NY6S1 (cheap-model client + redaction).
