## M11: Comprehensive full-stack smoke round-trips {#M11}

**Goal:** The smoke test calls every MCP tool + a hook, but reads are exercised
only against empty data, admin tools only checked for "not error," and hooks
verified via a counter. Add comprehensive **positive** full-stack round-trips
proving each MCP tool and each hook actually stores and retrieves correct data
end-to-end, driven through the real cheap model (tolerant assertions + retry,
clean skip without a model key). Hybrid structure: `scripts/smoke-test.sh` keeps
the Compose lifecycle + `MODEL` gate + teardown; comprehensive assertions move
into a new bun driver `scripts/smoke-assertions.ts` it invokes.

and M10 (`FR-ATBKZV` typed schema so array args drive cleanly + `FR-Z1W6ED` denoise,
whose meta-tool negative check this milestone validates end-to-end) shipped.
