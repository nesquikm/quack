# M10 — MCP schema fix + ambient ingestion denoise

## M10: MCP schema fix + ambient ingestion denoise {#M10}

**Goal:** Fix two defects found during the M-series install smoke on a real
schema-driven MCP client (`/private/tmp/quack-memory-findings.md`).
**(FR-ATBKZV)** MCP tools advertise an empty passthrough `inputSchema`, so
array-valued args (`search_memory.entities`, …) are mis-encoded by typed clients
and rejected as `invalid_args` — `search_memory` is undrivable. Advertise per-tool
typed `inputSchema` while keeping the handler-level `safeParse` as the validation
authority so `AC-WSFVNP.10` is preserved. **(FR-Z1W6ED)** Ingestion is over-eager:
the digester ingests the whole transcript, so casual conversation and tool chatter
become formal `Decision`s (a gaming opinion became a "SteamOS decision"). Keep
ambient capture but denoise — drop the agent's meta tool-activity at the client
hook, add an extractor decision-worthiness gate, and record `source` provenance.

**Prerequisites:** M2 (`FR-WSFVNP` — the admin/MCP tool surface + the `invalid_args`
contract ATBKZV must preserve), M5 (`FR-55S220` — the `.mcp.json` hook config path
the client drop sits beside), and the extractor pipeline (`FR-44QGKH` writer +
`source` field, `FR-4NY6S1` cheap-model client) all shipped.
