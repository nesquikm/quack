# M8 — Cross-project (sub-project) memory

## M8: Cross-project (sub-project) memory {#M8}

**Goal:** Let a connected set of repos (backend, frontend, shared packages) share one
Quack project and one token while every memory is tagged with — and filterable by —
its origin repo. Add a `source` sub-project dimension to the graph (server side), and
replace the direnv/`.envrc` config mechanism with a single `.mcp.json` written by an
adaptive `/quack:install` (client side). `project_id` stays the hard tenancy boundary;
`source` is a non-security provenance/filter label.

**Prerequisites:** M7 (Release-process hardening) shipped at v0.4.1.
