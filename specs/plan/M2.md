# Implementation Plan

## Milestone Order

M1 (Bootstrap) shipped pre-toolkit; M2 is the first feature milestone — multi-tenant auth, admin MCP surface, and Docker Compose deployment, sequenced by dependency.

## M2: Multi-tenant auth + Docker deployment {#M2}

**Goal:** Quack server boots from `docker compose up`, authenticates `(user, project)` tokens on both `/ingest` and `/mcp`, and an admin can manage users/projects/tokens via MCP tools — all without the memory plane (graph DB, `search_memory`, extractor) being wired up yet.

**Prerequisites:** M1 (Bootstrap) complete — green Bun gate, `specs/` populated, `auth.sqlite` schema agreed in `technical-spec.md` §2.

| BKPM28 | Docker Compose deployment | n/a |
