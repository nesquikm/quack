# Implementation Plan

## Milestone Order

M1 (Bootstrap), M2 (multi-tenant auth + admin MCP + server_status + Docker), M3 (memory plane — Neo4j + extraction + memory MCP tools + hook binary), and M4 (Claude Code marketplace plugin) all shipped. M5 adds a single MCP tool that lets the caller push free-form content into the knowledge base — routed through the same LLM digestion + Neo4j write path hooks already use.

## M5: add_memory MCP tool (LLM-digested writes via the existing extractor) {#M5}

**Goal:** A member-readable MCP tool `add_memory({ content })` that enqueues content for the existing FR-4NY6S1 extraction pipeline. Same redaction, same cheap-model digest, same MERGE templates, same cross-tenant isolation. Fire-and-forget — no status tracking, no new state, no new dependencies. The caller verifies via `search_memory` after a short delay, identical to the hook contract.

**Prerequisites:** M3 shipped (the queue + extractor pipeline from FR-4NY6S1 + the memory-plane MCP server registration from FR-DPY5GQ must exist). M2's auth middleware + admin tools must exist (the project_slug lookup uses `auth.sqlite`).
