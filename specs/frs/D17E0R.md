## Requirement

`scripts/smoke-test.sh` brings up the real Compose stack and *calls* every MCP
tool plus a hook, but its read-path coverage is shallow:

- the three read tools (`get_neighbors`, `path_between`, `recent_decisions`) are
  exercised only against **missing/empty** data;
- admin tools are only checked for "not an error";
- hooks are verified via the `accepted_total` **counter**, not by confirming the
  hooked content actually became retrievable.

The one genuine positive round-trip is `add_memory → (real model) → search_memory`.

This FR adds comprehensive full-stack **positive** round-trips proving each MCP
tool and each hook actually **stores and retrieves correct data end-to-end** —
`HTTP → MCP → Neo4j` and `hook → /ingest → extractor → Neo4j → read it back` —
**driven through the real cheap model** (the approved `/brainstorm` decision for
maximal realism).

**Hybrid structure (approved):** the bash `scripts/smoke-test.sh` keeps the
Compose lifecycle, the `MODEL` gate, and teardown; the comprehensive per-tool
assertions move into a new bun driver `scripts/smoke-assertions.ts` that the smoke
invokes. Because the extractor is non-deterministic, assertions are
**tolerant/semantic with retry/poll**, and the read tools use a
**discover-then-traverse** chain: seed via `add_memory`, poll `search_memory` to
find a real node id, then feed that id to `get_neighbors` / `path_between`.

## Technical Design

### Modules

- **`scripts/smoke-test.sh`** — replace the shallow read-tool checks (current `get_neighbors {missing}` / `path_between {a,b}` / `recent_decisions` shape checks). After minting `MTOK` (and when `MODEL=1`), call `bun "$ROOT/scripts/smoke-assertions.ts" "$URL" "$ADMIN" "$MTOK" "$SLUG"` and propagate its exit code. Keep the teardown trap, the `MODEL` gate, and the skip-when-docker-unreachable behavior.
- **`scripts/smoke-assertions.ts`** (new bun driver) — a `/mcp` call helper (bearer auth + `accept: application/json, text/event-stream`, parse `result.content[0].text`), a poll/retry helper, and tolerant matchers. Orchestrates: the discover→traverse read round-trip (AC.2); the hook round-trips (fire the real hook entry under `plugins/quack/hooks/_lib/entry/` or POST `/ingest` with known envelopes) plus the meta-tool denoise negative check (AC.3); the admin data-effect assertions with the destructive lifecycle ordered last (AC.4). Emits `ok`/`no` lines; exits non-zero on any failure. Uses bun's built-in `fetch` — no new dependency.
- **`.claude/skills/smoke-test/SKILL.md`** — refreshed description (AC.6).

### Data model

No schema change. Test/ops infrastructure only — no production code touched.

### Out of scope

- Deterministic / directly-seeded assertions (the real-model path was chosen for realism).
- CI automation of the smoke (it stays manual / skip-without-key; a separate concern).
- Any change to production behavior.

## Testing

This FR *is* test infrastructure, so its verification is the smoke run itself:
with `QUACK_MODEL_*` configured and Docker up, every positive round-trip passes;
with `MODEL` unset (or Docker down) the suite prints `SKIP` and exits 0. The gate
`bunx tsc --noEmit` must pass for the new TypeScript driver. Note `bun run test`
does **not** execute the smoke — it runs via the `smoke-test` skill or
`bash scripts/smoke-test.sh` directly.

## Notes

- Real-model + hybrid structure are the `/brainstorm` decisions: tolerant
  assertions + retry/poll + clean skip-without-key, consciously accepting model
  flakiness and token cost in exchange for exercising the real extraction path.
- `AC-D17E0R.3`'s meta-tool negative check doubles as an end-to-end regression
  guard for M10's ambient-ingestion denoise (FR-Z1W6ED) — proving noise stays out
  of the graph on the live stack, not just in unit tests.
- The destructive admin lifecycle (`remove_member` / `revoke_token` /
  `delete_project`) must run after the read + hook assertions, since it revokes
  the member token and deletes the project they depend on.
