## Requirement

Replace the M2 `POST /ingest` 204 stub with a real pipeline. `/ingest` validates the hook envelope and enqueues into a bounded in-process FIFO queue. A consumer task drains the queue with bounded concurrency, runs a **redaction pass** over the payload, calls the cheap-model API (OpenAI-compatible, `QUACK_MODEL_BASE_URL` + `QUACK_MODEL_API_KEY`) with a deterministic extraction prompt anchored to the v1 schema (5 nodes / 5 relations per `specs/technical-spec.md` §2), validates the response with Zod, normalizes entity names, and writes to Neo4j via parameterized `GraphAdapter` `MERGE` templates (idempotent). Failed extractions dead-letter to a local JSONL file. Counters (queue depth, accepted/dropped totals, extraction success/failure, redaction matches) feed `server_status` from FR-956DT2.

Redaction is **defense-in-depth**, not the only barrier — see also `specs/requirements.md` § Security/Abuse and § Redaction at hook layer (hook clients SHOULD scrub on the client side too).

## Acceptance Criteria

- AC-4NY6S1.1: `POST /ingest` body validated by Zod against `HookEnvelope = { kind: HookKind, payload: object, project_slug?: string, ts?: ISO8601 }`. `HookKind` is the literal-string union from FR-S2D0Z5 (hooks FR) — for M3 ship the set `"session_start" | "stop" | "post_tool_use"`. Validation failure ⇒ HTTP 400 `{ error: "invalid_envelope", path: [...] }` (the Zod issue path).
- AC-4NY6S1.2: Successful validation ⇒ enqueue. Response 202 `{ accepted: true, queued_at: ISO8601 }`. Queue full ⇒ 202 `{ accepted: false, reason: "queue_full", queued_at: null }` — still 202 because the hook contract is fire-and-forget; the server doesn't break the hook by 5xxing.
- AC-4NY6S1.3: In-process FIFO queue with capacity `QUACK_QUEUE_CAPACITY` (Zod env, default 10000). Implementation is a typed ring buffer in `src/extract/queue.ts`. Reads/writes are race-free under Bun's single-threaded runtime (consistent with FR-956DT2 AC.8). Queue depth is observable via `getDepth()`.
- AC-4NY6S1.4: Consumer task started at server startup. Concurrency cap via `QUACK_EXTRACTOR_CONCURRENCY` (Zod env, default 2). At most N extractions run in parallel. Cleanly drains on `SIGTERM`/`SIGINT`: finishes in-flight tasks; abandons queued items; logs `extractor_shutdown abandoned=<N>`.
- AC-4NY6S1.5: **Redaction pass.** `src/extract/redact.ts` applies a configurable deny-pattern list (`QUACK_REDACTION_PATTERNS` env, comma-separated extended-regex strings) to the hook payload BEFORE the outbound model POST. Default pattern set:
  - OpenAI key shape: `sk-[A-Za-z0-9]{20,}`
  - GitHub PAT shapes: `ghp_[A-Za-z0-9]{36,}`, `gho_[A-Za-z0-9]{36,}`, `ghs_[A-Za-z0-9]{36,}`
  - Slack tokens: `xox[abp]-[A-Za-z0-9\-]{10,}`
  - Generic Bearer: `Bearer\s+[\w._-]{20,}`
  - JWT: `eyJ[A-Za-z0-9_-]{20,}\.[A-Za-z0-9_-]{20,}\.[A-Za-z0-9_-]{20,}`
  - `.env`-style assignments: `(?:[A-Z][A-Z0-9_]+_(?:KEY|TOKEN|SECRET|PASSWORD))=[^\s]+`

  Matches are replaced with `«REDACTED»`. Patterns compile once at startup; deep-walk through nested objects + arrays + strings; no dynamic recompilation per request. Replacement count is reported back to the consumer (used for `redaction_match` counter increment).

- AC-4NY6S1.6: Cheap-model client uses the `openai` package with `baseURL: QUACK_MODEL_BASE_URL` + `apiKey: QUACK_MODEL_API_KEY`. Model name is `QUACK_MODEL_NAME` (Zod env, default `"gpt-4o-mini"`; overridable to any model the chosen provider serves). Request uses `response_format: { type: "json_schema", json_schema: { schema: <ExtractionResult JSON Schema>, strict: true } }` when the provider supports strict structured-output mode (OpenAI / Azure); falls back to `response_format: { type: "json_object" }` (most OpenAI-compatible endpoints) with the schema pasted into the system prompt. Probe order: try strict mode first; on `400 unsupported_response_format` retry with json_object mode and cache the per-baseURL result.

- AC-4NY6S1.7: Extraction prompt template (`src/extract/prompt.ts`) instructs the model to emit JSON conforming to `ExtractionResult`:

  ```ts
  ExtractionResult = {
    entities: Array<{ name: string, kind: string, aliases?: string[] }>,
    decisions: Array<{ summary: string, decided_at?: ISO8601, source_excerpt: string }>,
    files: Array<{ path: string, repo_root?: string }>,
    symbols: Array<{ name: string, file_path: string, kind: "function"|"class"|"type"|"variable"|"const" }>,
    feedbacks: Array<{ body: string, sentiment?: "positive"|"negative"|"neutral" }>,
    relations: Array<{
      type: "MENTIONS"|"DECIDED_BY"|"RELATED_TO"|"MODIFIES"|"FOLLOWS",
      from: { kind: NodeKind, name: string },
      to:   { kind: NodeKind, name: string },
      source_excerpt?: string
    }>
  }
  ```

  The prompt explicitly forbids invented labels and relation types and asks the model to omit rather than coerce when content doesn't fit. Empty arrays are valid outputs for sessions with nothing extractable.

- AC-4NY6S1.8: Server-side Zod parse on the model response. Parse failure ⇒ extraction fails → dead-letter + `extraction_failed` counter increment. Names with disallowed characters are coerced via `canonicalizeName(s)` (lowercase, ASCII; preserve `[a-z0-9 _\-.]`; strip everything else; collapse whitespace). `aliases` are also canonicalized; duplicate `(alias, name)` pairs deduplicated; an alias equal to the canonical name is dropped.

- AC-4NY6S1.9: Graph writes via `GraphAdapter` templates registered in `src/graph/templates/extract/`:
  - `upsert_entity` — `MERGE (e:Entity {project_id: $project_id, name: $name}) SET e.kind = $kind, e.created_at = coalesce(e.created_at, $now), e.aliases = apoc.coll.toSet(coalesce(e.aliases, []) + $aliases) RETURN e` (or `[…]` list union without APOC by using `CASE` + `COLLECT` if APOC is unavailable; pick the no-APOC path for Neo4j Community baseline).
  - `upsert_decision`, `upsert_file`, `upsert_symbol`, `upsert_feedback` — analogous `MERGE` patterns on the natural key per label (`File` on `path`, `Symbol` on `(name, file_id)` after resolving `file_id` from `path`).
  - `upsert_relation` — resolves `from` / `to` to node IDs via `MERGE` on the natural keys, then `MERGE (a)-[r:<type>]->(b) SET r.source_excerpt = $excerpt, r.created_at = coalesce(r.created_at, $now)`. Relation type is whitelisted in the template (5 allowed values) via a Cypher conditional `CASE` per-type sub-template, since Cypher doesn't allow parameterized relationship types.

  All templates declare `accessMode: 'WRITE'`. Re-running the same hook payload produces the same graph state modulo new aliases (deduplicated). Aliases extend `Entity.aliases: string[]` via set union.

- AC-4NY6S1.10: Counter integration — `src/metrics/counters.ts` is extended with:
  - `queue.depth: number` (live gauge — reads current queue size at snapshot time)
  - `queue.accepted_total: number` (incremented on enqueue)
  - `queue.dropped_full_total: number` (incremented on backpressure)
  - `incrementError("extraction_failed")` on parse failure / model error / write failure
  - `incrementError("queue_full")` on backpressure
  - `incrementError("redaction_match")` (info-level, fires when ≥ 1 redaction substitution occurred — visible in `errors.by_category` for ops monitoring)
  - `incrementError("db_error")` (already activated by FR-SFQDXR; extraction-write failures route through it)

  `server_status` from FR-956DT2 surfaces all these via its existing envelope; the `queue` block is no longer `null`.

- AC-4NY6S1.11: Dead letter — failed extractions append a JSONL line to `${QUACK_DATA_DIR}/dead-letters.jsonl`:

  ```json
  { "ts": "2026-05-13T10:23:14Z", "hook_kind": "stop", "project_id": 4,
    "error": { "kind": "zod_parse_failed", "message": "...", "model_response": "..." } }
  ```

  Rotates by size: env `QUACK_DEAD_LETTER_MAX_BYTES` (default 50MB) triggers rotation to `dead-letters.<ts>.jsonl` and re-opens the active file.

- AC-4NY6S1.12: Cross-tenant isolation — extracted nodes carry `project_id` from `ctx.project_id` (the `(user, project)` resolved by AuthMiddleware on `/ingest`). The writer layer overrides any model-supplied project hint with `ctx.project_id` before calling `GraphAdapter.run`. Integration test asserts this defense-in-depth override even when the model emits explicit `project_id` keys in its JSON output.

- AC-4NY6S1.13: Hook envelopes carrying `project_slug` that doesn't match the caller's `(user, project)` slug ⇒ HTTP 403 `{ error: "project_mismatch" }`. (Defense against a future hook client that knows multiple tokens — slug-spoofing must not enable cross-project ingestion.)

- AC-4NY6S1.14: End-to-end test in `src/extract/pipeline.test.ts`: seed a hook envelope, mock the cheap-model API to return a known `ExtractionResult` (a `Decision`, two `Entity` nodes, one `MENTIONS` relation), drive the consumer to drain, verify the graph contains the expected nodes/relations scoped to the caller's project, AND verify the redaction pass scrubbed a known secret pattern (e.g. an injected `sk-test123…` string) from the model-bound payload before the mock saw it.

## Technical Design

### Modules

- **`src/ingest/handler.ts`** — `POST /ingest`: envelope validation, slug-match check, enqueue, counter wiring, response shape.
- **`src/extract/queue.ts`** — typed FIFO ring buffer; `enqueue(envelope)`, `dequeue()`, `getDepth()`.
- **`src/extract/consumer.ts`** — drain loop; concurrency-capped `Promise.all` over a worker fleet; graceful shutdown.
- **`src/extract/redact.ts`** — pattern compilation at startup; `redact(value: unknown): { value: unknown, matchCount: number }` deep-walks payload (object / array / string).
- **`src/extract/client.ts`** — OpenAI SDK wrapper; structured-output mode + fallback; per-baseURL capability cache.
- **`src/extract/prompt.ts`** — system + user prompt templates; JSON schema embedded as a constant.
- **`src/extract/writer.ts`** — converts `ExtractionResult` → ordered `GraphAdapter.run` calls. `Entity` first, then `File` / `Symbol` / `Decision` / `Feedback`, then `Relation` (so endpoint nodes exist before relations). Single transaction per envelope via a Neo4j session.
- **`src/extract/dead_letter.ts`** — append-only JSONL writer with size-rotation.
- **`src/extract/canonicalize.ts`** — `canonicalizeName(s)` util used by writer.
- **`src/graph/templates/extract/{upsert_entity,upsert_decision,upsert_file,upsert_symbol,upsert_feedback,upsert_relation}.ts`** — six `MERGE` templates.
- **`src/metrics/counters.ts`** — extended with queue gauges + new error categories.
- **`src/shared/env.ts`** — extended with `QUACK_QUEUE_CAPACITY`, `QUACK_EXTRACTOR_CONCURRENCY`, `QUACK_REDACTION_PATTERNS`, `QUACK_MODEL_NAME`, `QUACK_DEAD_LETTER_MAX_BYTES`.
- **`src/server/index.ts`** — startup wires the consumer task; `SIGTERM` drains in-flight extractions; `/ingest` handler bound here.

### Dependencies added

- `openai` (`^5.x`, runtime; OpenAI-compatible client. Same SDK speaks Anthropic, Together, Ollama, local proxies, rubber-duck MCP's OpenAI endpoint).

### Out of scope here

- Hook scripts (client-side) — FR-S2D0Z5.
- Project-delete graph cleanup — FR-EDXH3X.
- Per-project cheap-model API keys (server-wide `QUACK_MODEL_API_KEY` only in v1).
- Extension labels beyond the v1 5+5 schema.
- Dead-letter replay tool — manual debugging surface only; replay is a future FR if needed.

## Notes

- **Redaction is defense-in-depth, not the only barrier.** Operators should also use `docker secrets` for `QUACK_MODEL_API_KEY` and avoid putting real secrets in hook-visible logs (the source `requirements.md` § Redaction is explicit about this; the hook client SHOULD scrub on the client side too — FR-S2D0Z5's responsibility).
- The `openai` package is chosen because every OpenAI-compatible endpoint speaks the same wire format — Anthropic (`https://api.anthropic.com/v1`), Together, Ollama (`http://localhost:11434/v1`), local proxies, and rubber-duck MCP's OpenAI-compat endpoint all interop. Switching providers = changing `QUACK_MODEL_BASE_URL` + `QUACK_MODEL_NAME`. No SDK switch required.
- `MERGE` patterns are the idempotency mechanism. Re-running the same hook payload (e.g., a `Stop` event re-posted by the hook client after a retry) produces the same graph state modulo `aliases` set union. This matters because hooks are best-effort and duplicates happen (network errors, hook-client retries).
- The `redaction_match` category is info-level, not failure-level — operators see it in `errors.by_category` to understand pattern frequency. A regularly-tripping pattern probably indicates the hook client should scrub source-side too.
- The transactional-per-envelope graph write is bounded — one Neo4j session per extracted envelope, all `MERGE` calls inside a single tx. Partial writes don't persist if any step fails. This keeps the graph internally consistent under failures.
- Future per-project model keys (out of scope): would add a `project_secrets.model_api_key_ciphertext` column to `auth.sqlite` plus a key-derivation step before each request. The current v1 contract is server-wide `QUACK_MODEL_API_KEY` only.
- The `apoc.coll.toSet` reference in AC.9's prose is illustrative; the canonical implementation uses a no-APOC fallback (`COLLECT` + `DISTINCT`) so Neo4j Community without APOC plugins works. Pick the no-APOC path; the spec language permits either.
